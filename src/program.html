CNUI 组件 AI 代码生成助手

核心实现流程

1. 链接MCP服务，获取可用工具列表，让AI知道能够调用哪些工具获取数据 2. 组装对话，把用户的提问加系统提示词一起发给大模型，告诉AI怎么写代码、按什么格式输出 3. 大模型推理，调用 qwen3-coder-plus 模型，AI 理解需求并生成代码 4. 工具调用，如果 AI 判断需要查组件 props/列表/demo/模板，就调用对应工具，生成更准确、符合组件规范的代码 5. 结果清洗与转换，去掉 markdown 代码块、按设备类型转换组件库，输出能直接在编辑器里运行的组件代码

一句话总结

对接大模型和组件工具链，根据用户输入自动生成符合设备类型（PC/移动）的 CNUI 组件代码，并完成代码清洗与组件库适配，让编辑器具备"一句话出代码"的能力

Cone CLI 智能 Agent 插件

核心实现流程

1. 识别项目类型：从 cone.conf.json 的 sceneType 自动判断是 MPA（多页应用）、SPA（单页应用）还是业务组件；识别不出来就弹出让用户选。 2. 识别编辑器类型：检查项目里是否已有 .cursor、.qoder、.aone_copilot 目录；没有就询问用户。 3. 复制规范模板：把插件里预置的 md-docs/<项目类型>/<编辑器类型> 目录里的规则文件（如编码规范、研发环境、低码/源码组件开发规则等）复制到项目根目录对应编辑器配置目录下。 4. 安全覆盖：如果目录已存在，只删除“模板里有的文件”，用户自己新增的文件会保留，不会误伤。

一句话总结

Cone CLI Agent 插件——为 Cone 脚手架体系提供 AI Agent 能力扩展，支持一键初始化团队 AI 编码规范、接入多智能体编码工具等能力。
